# DecodeLabs Frontend Development — Portfolio

A single-page frontend project built across three milestones for the DecodeLabs Industrial Training Kit (Batch 2026): static page structure, responsive layout, and JavaScript interactivity.

**Live demo:**
_add link after deploying (GitHub Pages / Netlify / Vercel)_

## What's in here

One HTML page, one stylesheet, one script file — no frameworks, no build tools, no dependencies.

```
decodelabs-project/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── main.js
└── images/
```

Open `index.html` in a browser. That's it.

---

## Project 1 — Static Webpage Design

Semantic HTML structure with external CSS only.

- Semantic tags throughout (`header`, `nav`, `main`, `article`, `aside`, `footer`)
- One `h1` per page, no skipped heading levels
- BEM-style class naming, no inline styles
- CSS Grid for page layout, Flexbox for component alignment
- Images with explicit width/height to prevent layout shift

## Project 2 — Responsive Web Layout

Mobile-first CSS with fluid scaling.

- Breakpoints at 480px / 768px / 1024px
- Fluid typography via `clamp()`
- Navigation built on the native HTML Popover API — no JS needed for the menu
- Touch targets at minimum 44×44px
- No zoom restrictions (`user-scalable` left untouched)

## Project 3 — Interactive Web Elements

Vanilla JavaScript, DOM manipulation, real interactivity.

- Dark/light mode toggle with `localStorage` persistence
- Counter widget with live state-based styling
- Character counter on live keyboard input
- Task list built with `createElement()` / `appendChild()`
- Accordion FAQ using `classList.toggle()`
- Scroll-reveal animations via `IntersectionObserver`
- `const`/`let` only, `textContent` over `innerHTML`, `js-` / `is-` class naming convention to keep styling and behavior decoupled

---

## Stack

HTML5, CSS3 (Custom Properties, Grid, Flexbox), vanilla JavaScript (ES6+). Fonts via Google Fonts (Syne, DM Sans, JetBrains Mono).
